# Benchmark

## Objetivo

Analizar plataformas existentes que organizan grandes volúmenes de información para identificar buenas prácticas de navegación, búsqueda y presentación de contenido que puedan aplicarse al proyecto One Piece Wiki.

---

# Referencia 1

## One Piece Wiki (Fandom)

### ¿Qué hace bien?

- Contiene información muy completa.
- Posee una gran cantidad de personajes y relaciones.
- Permite navegar entre personajes relacionados.

### Problemas encontrados

- Exceso de información en una sola página.
- Demasiada publicidad.
- Navegación compleja para usuarios nuevos.
- La información importante no siempre aparece primero.

### Aprendizajes

Nuestra aplicación deberá presentar únicamente la información más importante al inicio y dejar el contenido secundario para secciones adicionales.

---

# Referencia 2

## Marvel Database

### ¿Qué hace bien?

- Excelente organización visual.
- Navegación sencilla.
- Uso adecuado de imágenes.
- Relaciones claras entre personajes.

### Problemas encontrados

- Algunas páginas contienen demasiado texto.
- Existen múltiples niveles de navegación.

### Aprendizajes

Utilizar tarjetas visuales y relaciones directas entre personajes.

---

# Referencia 3

## Pokémon Database

### ¿Qué hace bien?

- Buscador muy rápido.
- Excelente sistema de filtros.
- Información organizada por categorías.
- Diseño limpio.

### Problemas encontrados

- Interfaz poco moderna.
- Escasa personalización visual.

### Aprendizajes

Implementar un buscador eficiente y filtros simples para mejorar la velocidad de consulta.

---

# Referencia 4

## IMDb

### ¿Qué hace bien?

- Información organizada por secciones.
- Excelente uso del espacio.
- Navegación consistente.
- Relación entre actores, películas y series.

### Problemas encontrados

- Gran cantidad de información puede resultar abrumadora.

### Aprendizajes

Dividir la información en bloques claramente diferenciados para mejorar la lectura.

---

# Oportunidades identificadas

Después del análisis se identifican las siguientes oportunidades para el proyecto:

- Reducir la cantidad de información mostrada inicialmente.
- Priorizar la navegación mediante imágenes y tarjetas.
- Facilitar la búsqueda de personajes.
- Relacionar personajes, tripulaciones, frutas del diablo y arcos mediante enlaces directos.
- Diseñar una interfaz moderna inspirada en el estilo visual de One Piece.

---

# Conclusión

Las referencias analizadas demuestran que existe una gran cantidad de información disponible sobre One Piece, pero también evidencian oportunidades para mejorar la experiencia de navegación mediante una interfaz más limpia, organizada y enfocada en la rapidez de consulta.
